# On-Chain Settlement — DONE (2026-06-11)

The settlement-level checks are implemented and **confirmed end-to-end against a
real chain** (local Anvil), with real funds moving and real transactions.

## What was built
- **`onchain/src/MockUSDC.sol`** — a faithful, self-contained EIP-3009 token:
  verifies the EIP-712 signature ON-CHAIN and tracks per-authorizer nonces (so
  replay / double-settle are enforced by the token, like real USDC). Its
  on-chain digest was proven byte-identical to the suite's off-chain signing.
- **`tools/onchain_facilitator.py`** — a real facilitator + resource server:
  off-chain verify (recipient/amount/time) + on-chain balance + settlement via
  `transferWithAuthorization` (real tx, real funds), plus `/verify` and `/settle`.
- **`tools/onchain_smoke.py`** — end-to-end smoke test (one valid payment).
- **RS-PAY group** (`checks/payment.py`): RS-PAY-001..004 + RS-SEC-001 (replay).
- **FA-SET group** (`checks/facilitator.py`): FA-SET-001/002/003 (settle / invalid / double-settle).
- CLI: `check --pay [--rpc-url]`, `facilitator --settle`.
